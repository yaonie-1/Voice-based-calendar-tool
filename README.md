# 语音版日历工具

这是我做的一个语音日历小工具。主要想解决的问题是：有时候临时想到一件事，但是打开日历、选日期、填时间比较麻烦，所以我做了一个可以直接说话来添加日程的页面。

目前这个项目是前端网页版本，数据保存在浏览器本地。打开页面后，可以用语音或者文字指令来添加、查看、删除日程，也可以设置提前提醒。

## 能做什么

现在主要做了这几件事：

- 语音添加日程，比如“明天下午三点提醒我开会”
- 语音查看日程，比如“查看今天的日程”“查看本周日程”
- 语音删除日程，比如“删除开会”
- 到时间后弹出浏览器提醒，并用语音读出来
- 浏览器语音识别不好用时，也可以直接输入文字指令
- 也保留了普通表单添加，方便测试

## 测试过的指令

```text
明天下午三点提醒我开会
添加今天晚上八点健身，提前十分钟提醒
下周一上午九点安排项目讨论
查看今天的日程
查看明天的安排
查看本周日程
查看全部日程
删除开会
取消健身
```

目前时间解析只覆盖了一些常见说法。太复杂的表达，比如“下个月第一个周五下午三点”，暂时没有做。

## 技术说明

这个项目没有用 Vue、React 之类的框架，也没有装第三方 npm 包。主要是为了让项目简单一点，评委或者同学拉下来以后直接能跑。

| 部分 | 用到的内容 |
| --- | --- |
| 页面 | HTML、CSS |
| 主要逻辑 | JavaScript ES Module |
| 语音识别 | 浏览器 Web Speech API |
| 语音播报 | SpeechSynthesis API |
| 日程保存 | localStorage |
| 提醒 | Notification API + 定时检查 |

需要注意的是，Web Speech API 的支持情况和浏览器有关。我自己主要按 Chrome / Edge 的情况来做。如果浏览器不支持语音识别，可以先用文字输入测试。

## 怎么运行

### 方法一：直接打开

可以直接双击 `index.html` 打开页面。文字指令和基本页面功能可以测试。

### 方法二：用本地服务打开

更推荐这个方法，因为浏览器权限相关功能会稳定一点。

```bash
python -m http.server 5173
```

然后访问：

```text
http://localhost:5173
```

如果本机没有 Python，也可以用 VS Code 的 Live Server 插件打开。

## 简单测试流程

1. 打开页面。
2. 点击“按下说话”。
3. 说“明天下午三点提醒我开会”。
4. 看列表里有没有新增“开会”。
5. 再说“查看明天的日程”。
6. 看页面和语音反馈是否正确。
7. 最后说“删除开会”，检查是否删除。

如果麦克风权限没开，或者浏览器不支持语音识别，就把这些句子输入到文本框里测试。

## 关于持续提交

题目要求不能最后一天一次性导入全部代码，所以这个项目不要一次性全提交。可以按 `docs/05-pr-delivery-plan.md` 里面的顺序拆成几个 PR 来做。

我建议的顺序是：

1. 先提交基础页面。
2. 再提交本地日程存储。
3. 再做手动添加和删除。
4. 再接语音识别和播报。
5. 再做指令解析。
6. 最后补提醒功能和文档。

每个 PR 描述里写清楚这次改了什么、怎么测试，不要空着。

## 目前不足

这个版本还不是完整的商业日历，只是一个可以演示主要思路的网页工具。现在主要有这些限制：

- 页面关闭后不会继续提醒。
- 没有接入系统日历或者手机日历。
- 自然语言解析是规则写的，复杂句子不一定能识别。
- 语音识别效果受浏览器和麦克风影响。

后面如果继续做，可以考虑接入真实日历账号，或者把提醒放到后端 / Service Worker 里处理。
